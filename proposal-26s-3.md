# 鸿蒙平板数字笔记本（Canvas）应用项目 Proposal

## 一、项目概述

我们团队计划开发一个面向 HarmonyOS 平板 的数字笔记本应用。该应用以“Canvas”为核心交互形式，重点支持用户在无限或大尺寸画布上进行手写记录、绘图表达和笔记整理，适用于学习、会议记录、思维导图草稿、灵感记录等场景。

与传统文档阅读类应用不同，本项目聚焦于数字笔记本本身的创建、编辑、组织、导入与导出。用户可以在应用中创建多个笔记本，自由书写、绘图、插入基础元素和图片，并通过页面管理、模板背景和 PDF 导入等方式对内容进行整理。由于项目面向鸿蒙平板，系统将重点考虑大屏交互、手写笔体验、低延迟书写以及本地数据持久化。

## 二、初步需求分析

### 1. 功能需求（Functional Requirements）

本系统拟包含以下 7 个较为独立且具有代表性的高层功能。

### 功能一：数字笔记本创建与管理

用户可以在应用中创建多个数字笔记本，用于区分课程、项目、会议或个人主题。系统需要支持基础的笔记本管理能力。

具体包括：

- 新建笔记本
- 重命名笔记本
- 删除笔记本
- 查看笔记本列表
- 按创建时间、最近修改时间或名称排序
- 点击进入指定笔记本

### 功能二：Canvas 手写与绘图

这是系统最核心的功能。用户能够在画布上使用手写笔自由书写和绘图，获得接近真实纸张的记录体验。

具体包括：

- 在画布上自由书写
- 支持不同笔刷类型，如钢笔、铅笔、荧光笔
- 支持调整笔的粗细
- 支持不同颜色书写
- 支持橡皮擦功能
- 支持撤销与重做
- 支持手写内容实时显示并自动保存

### 功能三：页面与画布管理

数字笔记本通常由多个页面组成，因此系统需要支持页面级别的组织与操作。同时，画布可以支持较大书写区域，以满足连续记录需求。

具体包括：

- 在笔记本中新增页面
- 删除页面
- 页面顺序调整
- 页面缩略图预览
- 页面切换
- 支持画布缩放、拖动画布查看不同区域
- 支持不同页面模板，如空白、横线、方格、点阵
- 支持设置或切换纸张背景

### 功能四：基础元素插入与编辑

为了增强表达能力，用户除了手写外，还应能够插入一些基础元素和图片，提高笔记的丰富性和结构性。

具体包括：

- 插入文本框
- 插入简单图形，如直线、矩形、圆形
- 插入图片
- 移动、缩放和删除已插入元素
- 在手写、图形和图片内容之间进行简单排版

### 功能五：导出与分享

用户应能够将自己的数字笔记导出，用于备份、打印、提交作业或分享给他人。

具体包括：

- 将单页或整个笔记本导出为 PDF
- 将页面导出为图片
- 将导出结果保存到本地
- 通过系统支持的方式分享导出文件

### 功能六：应用界面设计与交互优化

为了提升用户体验，系统需要具备清晰、美观且易于操作的界面设计，并针对 HarmonyOS 平板设备进行专门优化，使用户能够高效完成笔记记录与管理。

具体包括：

- 设计应用整体页面结构，包括首页、笔记本列表页、编辑页及页面之间的跳转关系
- 设计编辑页工具栏界面，包括画笔、橡皮擦、撤销、重做、页面切换等常用功能入口
- 统一应用视觉风格，包括按钮、图标、颜色、字体和间距
- 优化平板端交互体验，确保页面在大屏设备上显示合理，操作流畅自然
- 实现页面切换与交互反馈，包括页面跳转、按钮点击反馈、弹窗提示等细节

### 功能七：导入 PDF 文档

为了满足课堂资料整理、讲义阅读和参考文档记录等场景，系统应支持将本地 PDF 文档导入到应用中，并与笔记内容结合使用。

具体包括：

- 从本地导入 PDF 文件
- 解析 PDF 页面并生成基础缩略图信息
- 在应用内浏览和切换 PDF 页面
- 将 PDF 页面导入到指定笔记本中进行统一管理
- 支持在导入后的 PDF 页面上进行手写批注或补充记录
- 本地保存导入文档及相关笔记数据

### 2. 非功能需求（Non-functional Requirements）

#### 易用性

系统界面应简洁直观，符合平板和手写笔的使用习惯。常用工具如画笔、橡皮、撤销重做、页面切换等应能快速访问，降低学习成本，提高书写效率。

#### 性能

应用应尽量降低手写延迟，保证用户书写流畅。画布缩放、页面切换、PDF 导入与笔记保存应保持较快响应，避免卡顿，提升整体使用体验。

#### 可靠性

笔记数据是用户的重要内容，系统应具备较高的稳定性和可靠的数据保存机制，避免因应用崩溃、异常退出或导入过程异常导致内容丢失。

#### 安全性

用户创建的笔记、导入的 PDF 资料和相关元数据应保存在本地安全环境中，避免未授权访问。若后续扩展云同步功能，也应考虑传输和存储安全。

#### 可维护性

系统应采用清晰的模块化设计，例如将画布渲染、输入处理、页面管理、PDF 导入、数据存储等模块分离，方便团队协作、后续维护与功能扩展。

#### 兼容性

系统需适配 HarmonyOS 平板设备，支持常见屏幕尺寸与分辨率，并对手写笔、触控输入和常见 PDF 文件保持较好的兼容性。

### 3. 技术需求（Technical Requirements）

#### 运行环境

系统应能够在 HarmonyOS 平板真机或模拟器环境中运行，并支持手写笔和触控输入。

#### 开发技术栈

- 目标平台：HarmonyOS 平板
- 开发工具：DevEco Studio
- 版本管理：Git + GitHub
- 项目管理工具：GitHub Projects
- 前端界面：ArkUI
- 开发语言：ArkTS
- 画布绘制：基于 HarmonyOS 提供的绘制能力或可兼容的 Canvas 方案
- PDF 处理：基于系统文件选择能力和可兼容的 PDF 解析方案实现导入与页面展示
- 本地数据存储：Preferences、本地文件系统或轻量级数据库
- 状态管理：用于维护笔记本、页面、画笔设置和用户操作状态
- 测试方式：手动功能测试 + 必要的单元测试/集成测试

#### 执行要求

项目交付应覆盖数字笔记本创建、Canvas 书写、页面管理、基础元素与图片插入、PDF 导入及导出等主要功能。

### 4. 数据需求（Data Requirements）

本系统的数据主要来源于用户在应用中的输入和系统内部生成的信息，不依赖大规模外部数据集。

#### （1）笔记本数据

包括笔记本名称、创建时间、修改时间等信息。  
数据来源：用户在应用中创建和编辑。  
获取方式：系统在用户操作时自动生成并本地保存。

#### （2）页面数据

包括每个笔记本中的页面顺序、页面编号、缩略图信息、页面模板和纸张背景类型等。  
数据来源：用户新建、删除、调整页面或切换模板时产生。  
获取方式：由系统维护并存入本地数据结构。

#### （3）手写笔迹数据

包括用户在画布上的轨迹坐标、颜色、笔刷类型、粗细等。  
数据来源：用户通过手写笔或触控输入产生。  
获取方式：通过输入事件实时采集并本地保存。

#### （4）图形、文本与图片元素数据

包括文本框内容、图形类型、图片路径、位置、大小等。  
数据来源：用户插入和编辑基础元素或图片时生成。  
获取方式：系统记录元素属性并存储在本地。

#### （5）导入 PDF 文档数据

包括 PDF 文件名称、页数、页面缩略图、导入时间、本地存储路径及所属笔记本关联信息等。  
数据来源：用户从设备本地选择并导入 PDF 文件。  
获取方式：系统读取 PDF 文件信息，并转换为可展示和可管理的页面数据后本地保存。

#### （6）元数据与使用记录

包括最近打开时间、导出历史、导入历史、用户设置等。  
数据来源：用户日常使用过程。  
获取方式：由系统自动记录并更新。

#### （7）测试数据

开发阶段需要准备一些示例笔记本、示例页面、示例 PDF 文件和手写输入数据，用于验证画布性能和功能完整性。  
数据来源：团队自行构造的测试数据。  
获取方式：手动创建和导入测试环境。

总体来看，本项目的数据需求较为清晰，主要依赖用户输入和本地持久化，数据获取成本较低，实现路径明确。


## 三、目标用户与使用场景

本系统的目标用户主要包括：

- 需要在课堂上手写记录的学生
- 需要快速整理会议纪要的办公人员
- 需要绘制草图、流程图或思维导图草稿的用户
- 希望在平板上进行无纸化记录的普通用户
- 需要结合讲义或资料 PDF 进行阅读与补充记录的用户

典型使用场景包括：

- 学生在课堂上创建课程笔记本并分页面记录知识点
- 用户在会议中快速手写记录讨论内容
- 用户在画布中整理思路、绘制草图或简单结构图
- 用户将整理好的笔记导出为 PDF 或图片进行分享和提交
- 用户导入课程讲义或会议资料 PDF，并在对应页面上补充手写笔记和重点标记

## 五、总结

综上所述，我们团队拟开发一款面向 HarmonyOS 平板 的数字笔记本（Canvas）应用。该系统以手写输入和画布交互为核心，围绕笔记本创建、Canvas 书写、页面管理、模板背景、基础元素与图片编辑、PDF 导入以及导出分享等功能展开设计。

从需求明确性、技术实现路径和团队协作角度来看，该项目具有较强的实用价值和可实现性，并能够通过后续开发逐步完善。
