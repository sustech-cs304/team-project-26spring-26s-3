# 鸿蒙平板数字笔记本项目需求与一期落地方案

## 1. 项目目标

实现一个可在 `DevEco Studio` 中直接打开和运行的 HarmonyOS 平板应用，技术栈采用：

- `ArkTS`
- `ArkUI`

应用定位为数字笔记本，不做 PDF 管理或批注，核心是基于画布的手写记录与页面组织。

## 2. 已确认范围

### 2.1 保留功能

1. 数字笔记本创建与管理
2. Canvas 手写与绘图
3. 页面与画布管理
4. 基础元素插入与编辑
5. 导出与分享

### 2.2 明确不做

- 不实现“分类 / 标签 / 搜索检索”
- 不实现云同步
- 不做多人协作
- 不做 PDF 导入后批注

## 3. 一期 MVP 范围

为了保证项目可以稳定落地，一期先实现最核心闭环。

### 3.1 一期必须完成

1. 应用基础工程可在 `DevEco Studio` 正常打开、编译、运行
2. 笔记本列表页
3. 新建 / 重命名 / 删除笔记本
4. 进入笔记本后查看页面列表
5. 新建 / 删除 / 切换页面
6. 页面内画布手写
7. 画笔颜色、粗细切换
8. 橡皮擦
9. 撤销 / 重做
10. 自动保存到本地
11. 重新进入应用后可恢复笔记内容

### 3.2 一期暂缓

## 5. 页面结构

建议先按以下页面组织应用。

### 5.1 `NotebookListPage`

用途：

- 显示所有笔记本
- 支持新建、重命名、删除、排序

主要信息：

- 笔记本名称
- 创建时间
- 最近修改时间
- 页面数量

### 5.2 `NotebookDetailPage`

用途：

- 展示某个笔记本下的页面列表
- 支持创建页面、切换页面、删除页面

一期可先做成左侧页面列表 + 右侧当前页面入口，或直接点击页面进入编辑页。

### 5.3 `EditorPage`

用途：

- 核心编辑页面
- 承载 Canvas 画布与工具栏

工具栏一期建议包含：

- 画笔
- 橡皮擦
- 颜色选择
- 粗细选择
- 撤销
- 重做
- 返回

## 7. 存储方案建议

一期以简单、稳定、易调试为主。

### 7.1 推荐方案

- `Preferences`：保存轻量配置与最近状态
- 本地文件：保存笔记本、页面、笔迹数据 JSON

原因：

- 实现成本低
- 容易调试
- 便于在课程项目中展示数据结构
- 足以支撑一期 MVP

### 7.2 暂不建议

- 一开始就上复杂数据库

原因：

- 当前项目数据规模不大
- 先保证编辑闭环，再做存储升级更稳妥

## 8. 模块划分

建议在 HarmonyOS 工程中按职责拆分：

### 8.1 `pages`

- 页面级 UI
- 例如列表页、详情页、编辑页

### 8.2 `components`

- 可复用 UI 组件
- 例如工具栏、颜色选择器、页面卡片

### 8.3 `models`

- 数据模型定义

### 8.4 `store`

- 状态管理
- 管理当前笔记本、当前页面、当前工具状态

### 8.5 `services`

- 数据读写服务
- 笔记本、页面、笔迹的本地持久化

### 8.6 `canvas`

- 画布绘制逻辑
- 输入事件处理
- 笔迹渲染
- 撤销重做栈
